# 插件与权限

本章介绍 EasyBot SDK 的插件系统和权限管理功能，包括命令注册、预处理器和权限校验。

---

## 插件系统

EasyBot SDK 提供两种命令注册方式：

### 1. Plugins 类方法装饰器（推荐用于插件文件）

**模块**: `easybot.plugins`
**导入**: `from easybot import Plugins`

`Plugins` 类提供类方法装饰器，可直接通过类名调用，无需实例化。命令会在模块导入时自动注册到类变量中，全局共享。

#### 注册命令

```python
from easybot import Plugins, CommandValidScenes

@Plugins.on_command(command="/hello")
async def cmd_hello(msg):
    await msg.reply("Hello!")

@Plugins.on_command(regex=r"^计算\s+(.+)$")
async def cmd_calc(msg):
    expression = msg.treated_msg[0]
    result = eval(expression, {"__builtins__": {}})
    await msg.reply(f"= {result}")
```

#### 注册预处理器

```python
from easybot import Plugins, CommandValidScenes

@Plugins.before_command(valid_scenes=CommandValidScenes.ALL)
async def preprocess(msg):
    print(f"收到消息: {msg.content}")
    return True
```

### 2. Bot 实例方法装饰器（推荐用于单文件程序）

使用 `@bot.on_command` 和 `@bot.before_command` 装饰器，适用于单文件程序的快速开发。

#### 注册命令

```python
@bot.on_command(command="/ping")
async def cmd_ping(msg):
    await msg.reply("Pong!")

@bot.on_command(command=["/help", "帮助"])
async def cmd_help(msg):
    await msg.reply("可用命令:\n/ping - 测试连接")
```

#### 注册预处理器

```python
@bot.before_command()
async def preprocess(msg):
    print(f"收到消息: {msg.content}")
    return True
```

### 命令动态管理 API

Plugins 类提供了完整的命令动态管理功能：

#### 获取所有命令（包括禁用的）

```python
all_commands = Plugins.get_all_commands()
for cmd in all_commands:
    status = "✅" if cmd.enabled else "❌"
    print(f"{status} {cmd.func.__name__}")
```

#### 查找命令对象

```python
cmd = Plugins.find_command("cmd_hello")
if cmd:
    print(f"找到命令: {cmd.func.__name__}")
```

#### 启用/禁用命令

```python
# 禁用命令
Plugins.disable_command("cmd_hello")

# 启用命令
Plugins.enable_command("cmd_hello")
```

#### 检查命令是否启用

```python
if Plugins.is_command_enabled("cmd_hello"):
    print("命令已启用")
else:
    print("命令已禁用或不存在")
```

#### 移除命令

```python
Plugins.remove_command("cmd_hello")
```

### 装饰器参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `command` | `str \| list[str]` | — | 文本命令列表，与 regex 二选一 |
| `regex` | `Pattern \| str` | — | 正则表达式，与 command 二选一 |
| `is_treat` | `bool` | `True` | 是否处理消息内容 |
| `is_require_at` | `bool` | `False` | 是否要求 @机器人 |
| `is_short_circuit` | `bool` | `True` | 触发后是否短路 |
| `is_custom_short_circuit` | `bool` | `False` | 回调返回 True 时短路 |
| `is_require_admin` | `bool` | `False` | 是否要求频道管理员 |
| `admin_error_msg` | `str \| None` | `None` | 管理员权限不足提示 |
| `valid_scenes` | `CommandValidScenes` | ALL | 有效场景 |
| `enabled` | `bool` | `True` | 是否启用 |
| `is_require_bot_admin` | `bool` | `False` | 是否要求机器人管理员 |
| `bot_admin_error_msg` | `str \| None` | `None` | 机器人管理员权限不足提示 |

**参数映射说明**:

装饰器参数与 `BotCommandObject` 内部属性名存在以下映射关系：

| 装饰器参数 | 内部属性 | 说明 |
|-----------|---------|------|
| `is_treat` | `treat` | 是否处理消息 |
| `is_require_at` | `at` | 是否要求 @机器人 |
| `is_short_circuit` | `short_circuit` | 是否短路 |
| `is_require_admin` | `admin` | 是否要求频道管理员 |
| `is_require_bot_admin` | `is_require_bot_admin` | 是否要求机器人管理员 |

### valid_scenes 有效场景

```python
from easybot import CommandValidScenes

# 使用 Plugins 类装饰器
@Plugins.on_command(command="/guild_cmd", valid_scenes=CommandValidScenes.GUILD)

# 使用 Bot 实例装饰器
@bot.on_command(command="/guild_cmd", valid_scenes=CommandValidScenes.GUILD)
```

**场景常量**:

| 常量 | 值 | 说明 |
|------|-----|------|
| `GUILD` | 1 | 频道消息 |
| `DM` | 2 | 频道私信 |
| `GROUP` | 4 | QQ 群聊消息 |
| `C2C` | 8 | QQ 单聊消息 |
| `ALL` | 15 | 全部场景 |

### 消息对象 treated_msg 属性

命令匹配后，消息对象的 `treated_msg` 字段会被更新：

| 匹配模式 | `treated_msg` 值 |
|---------|-----------------|
| 文本命令 `is_treat=True` | 去除命令前缀后的字符串 |
| 文本命令 `is_treat=False` | 原始消息内容 |
| 正则命令 | 正则捕获组元组 `tuple` |

```python
@Plugins.on_command(command="/echo")
async def echo(msg):
    # 用户输入 "/echo hello world"
    # msg.treated_msg == "hello world"
    pass

@bot.on_command(regex=r"^计算\s+(.+)$")
async def calc(msg):
    # 用户输入 "计算 1+1"
    # msg.treated_msg == ("1+1",)
    expression = msg.treated_msg[0]
    pass
```

## 预处理器

预处理器在命令匹配前执行，可用于日志记录、消息过滤、自定义权限检查等。

### 使用 Plugins 类装饰器

```python
@Plugins.before_command()
async def preprocess(msg):
    print(f"收到消息: {msg.content}")
    # 返回 False 可跳过后续命令匹配
    return True
```

### 使用 Bot 实例装饰器

```python
@bot.before_command()
async def preprocess(msg):
    bot.logger.info(f"收到消息: {msg.content}")
    return True
```

## 自动加载插件

Bot 构造函数支持自动加载插件目录：

```python
bot = Bot(
    app_id="xxx",
    app_secret="xxx",
    auto_load_plugins=True,      # 启用自动加载
    plugins_dir="plugins",       # 插件目录路径
    plugins_recursive=False,     # 是否递归扫描子目录
)
```

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `auto_load_plugins` | `bool` | `False` | 是否自动加载插件目录 |
| `plugins_dir` | `str` | `"plugins"` | 插件目录路径 |
| `plugins_recursive` | `bool` | `False` | 是否递归扫描子目录 |

### 插件编写方式

#### 方式一：类方法装饰器（推荐）

直接使用 `@Plugins.on_command()` 装饰器，命令在模块导入时自动注册：

```python
# plugins/admin.py

from easybot import Plugins, CommandValidScenes

@Plugins.on_command(
    command="/admin",
    is_require_admin=True,
    admin_error_msg="此指令仅管理员可用",
)
async def admin_cmd(msg):
    await msg.reply("欢迎管理员！")

@Plugins.on_command(
    command="/ping",
    valid_scenes=CommandValidScenes.GUILD | CommandValidScenes.GROUP,
)
async def ping_cmd(msg):
    await msg.reply("Pong!")
```

#### 方式二：register 函数

定义 `register(bot)` 函数，可在函数内访问 bot 实例：

```python
# plugins/hello.py

from easybot import Plugins, Bot

def register(bot: Bot):
    # 可访问 bot 实例，如获取配置、日志等
    bot.logger.info("加载 hello 插件")

    @Plugins.on_command(command="/hello")
    async def hello(msg):
        await msg.reply("Hello!")
```

### 插件目录结构示例

```
plugins/
├── admin.py          # 管理员插件
├── weather.py        # 天气查询插件
├── translate.py      # 翻译插件
└── utils/            # 子目录（需开启 plugins_recursive）
    └── helpers.py    # 辅助工具插件
```

## BotAdminManager

**模块**: `easybot.plugins`
**导入**: `from easybot import BotAdminManager`

BotAdminManager 用于管理机器人管理员（全局超管）。管理员数据自动持久化到 YAML 文件（默认路径 `sdk_data/bot_admins.yaml`），启动时自动加载，增删操作后自动保存。

### 管理方法

```python
manager = BotAdminManager()

# 批量设置管理员列表（合并式，与现有数据取并集，立即持久化到 YAML 文件）
manager.bot_admins = ["user_id_1", "user_id_2", "user_id_3"]

# 增量添加/移除（每次操作后自动持久化）
manager.add_admin("user_id_4")
manager.remove_admin("user_id_1")

if manager.is_admin("user_id_1"):
    print("是管理员")

admins = manager.get_all_admins()

manager.clear_admins()
```

### bot_admins 属性

`bot_admins` 是一个可读写属性，用于批量设置或获取管理员列表：

```python
# 获取当前管理员列表
current_admins = manager.bot_admins  # 返回 list[str]

# 设置管理员列表（合并式，与现有数据取并集，不会丢失已有管理员）
manager.bot_admins = ["new_id_1", "new_id_2"]
```

### 自定义存储目录

可通过 `data_dir` 参数指定存储目录（默认为当前工作目录下的 `sdk_data/`）：

```python
# 使用自定义存储目录
manager = BotAdminManager(data_dir="/path/to/my_data")
```

### 支持的操作符

```python
if "user_id" in manager:
    pass

len(manager)
```

### 使用方式

BotAdminManager 支持两种使用方式：

#### 方式一：直接实例化（**推荐**）

独立于 `Bot` 实例，在任意位置创建和管理管理员。适合将配置与机器人启动解耦的场景。

```python
from easybot import BotAdminManager

# 独立实例化（自动从 sdk_data/bot_admins.yaml 加载已有数据）
admin_mgr = BotAdminManager()

# 设置管理员（合并式，立即持久化）
admin_mgr.bot_admins = ["admin_1", "admin_2"]

# 增量操作（每次自动持久化）
admin_mgr.add_admin("admin_3")
admin_mgr.remove_admin("admin_1")

# 查询
if admin_mgr.is_admin("user_id"):
    print("是管理员")
```

> **推荐理由**：与 `Bot` 完全解耦，可在单独的配置文件中管理；`Bot` 内部的权限检查会读取同一个 YAML 文件，数据天然共享。

#### 方式二：通过 Bot 实例访问

`Bot` 类内置了 `bot_admin_manager` 属性，返回一个 `BotAdminManager` 实例。

```python
from easybot import Bot

bot = Bot(app_id="xxx", app_secret="xxx")

# 通过属性访问，操作与方式一完全相同
bot.bot_admin_manager.bot_admins = ["admin_1", "admin_2"]
bot.bot_admin_manager.add_admin("admin_3")
bot.bot_admin_manager.remove_admin("admin_1")

if bot.bot_admin_manager.is_admin("user_id"):
    print("是管理员")
```

> **适用场景**：单文件程序中快速访问，或需要在命令回调中引用时使用。

#### 两种方式的对比

| 维度 | 方式一：直接实例化 | 方式二：Bot 实例访问 |
|------|-------------------|-------------------|
| **依赖关系** | 不需要 `Bot` 实例 | 依赖 `Bot` 实例 |
| **使用位置** | 任意模块/文件 | 需要能访问到 `bot` 变量 |
| **数据共享** | ✅ 同一 YAML 文件自动共享 | ✅ 同一实例 |
| **推荐场景** | 配置文件、多模块项目、插件开发 | 单文件程序、命令回调内部 |
| **持久化** | ✅ 自动 | ✅ 自动 |

#### 数据共享原理

无论使用哪种方式，最终都读写同一个持久化文件 `sdk_data/bot_admins.yaml`：

```python
# config.py — 用方式一设置管理员
from easybot import BotAdminManager
admin_mgr = BotAdminManager()
admin_mgr.bot_admins = ["owner_id"]     # 写入 sdk_data/bot_admins.yaml

# main.py — Bot 启动后自动加载同一份数据
from easybot import Bot
bot = Bot(app_id="...", app_secret="...")

@bot.on_command(command="/shutdown", is_require_bot_admin=True)
async def shutdown(msg):
    # 权限检查时自动从 sdk_data/bot_admins.yaml 读取，能识别 owner_id
    await msg.reply("正在关闭...")
```

## 权限控制

EasyBot 采用双层权限体系：

| 权限类型 | 参数 | 校验方式 | 适用场景 |
|---------|------|---------|---------|
| **频道管理员** | `is_require_admin` | QQ 平台身份组 | 频道内管理操作 |
| **机器人管理员** | `is_require_bot_admin` | 自定义白名单 | 全局业务管控 |

### 频道管理员权限

```python
# 使用 Plugins 类装饰器
@Plugins.on_command(
    command="/ban",
    is_require_admin=True,
    admin_error_msg="⛔ 需要频道管理员权限"
)
async def ban_user(msg):
    pass

# 使用 Bot 实例装饰器
@bot.on_command(
    command="/ban",
    is_require_admin=True,
    admin_error_msg="⛔ 需要频道管理员权限"
)
async def ban_user(msg):
    pass
```

**注意**: 此权限依赖 QQ 平台的 `member.roles` 数据，仅在**频道消息**场景有效。

### 机器人管理员权限

```python
# 使用 Plugins 类装饰器
@Plugins.on_command(
    command="/shutdown",
    is_require_bot_admin=True,
    bot_admin_error_msg="🚫 仅机器人管理员可操作"
)
async def shutdown(msg):
    pass

# 使用 Bot 实例装饰器
@bot.on_command(
    command="/shutdown",
    is_require_bot_admin=True,
    bot_admin_error_msg="🚫 仅机器人管理员可操作"
)
async def shutdown(msg):
    pass
```

### 组合使用

```python
@Plugins.on_command(
    command="/config",
    is_require_admin=True,
    is_require_bot_admin=True,
)
async def config(msg):
    pass
```

## 权限校验流程

权限校验在命令匹配过程中进行，按以下顺序检查：

```
收到消息
    │
    ▼
遍历已注册命令
    │
    ├─ ① at 检查（如果设置了 is_require_at）
    │      └─ 不包含 @机器人 → 跳过此命令
    │
    ├─ ② 频道管理员检查（如果设置了 is_require_admin）
    │      ├─ 权限不足 + admin_error_msg 有值 → 回复消息并短路
    │      └─ 权限不足 + admin_error_msg 为 None → 静默跳过
    │
    ├─ ③ 机器人管理员检查（如果设置了 is_require_bot_admin）
    │      ├─ 权限不足 + bot_admin_error_msg 有值 → 回复消息并短路
    │      └─ 权限不足 + bot_admin_error_msg 为 None → 静默跳过
    │
    └─ ④ 命令匹配 + 执行回调
```

**关键行为**:
- 权限检查不通过时，**跳过当前命令并继续匹配后续命令**，而非结束整个消息处理
- 设置错误消息时，命令会回复提示并短路（不再匹配后续命令）
- 不设置错误消息时，命令静默跳过（对用户不可见）

## 最佳实践

### 1. 敏感操作使用机器人管理员权限

```python
@Plugins.on_command(command="/reload", is_require_bot_admin=True)
async def reload_config(msg):
    # 重新加载配置
    pass
```

### 2. 管理操作使用频道管理员权限

```python
@bot.on_command(command="/mute", is_require_admin=True)
async def mute_user(msg):
    # 禁言用户
    pass
```

### 3. 使用预处理器实现复杂逻辑

```python
@Plugins.before_command()
async def rate_limit(msg):
    user_id = msg.author.id
    if is_rate_limited(user_id):
        return False  # 跳过命令匹配
    return True
```

### 4. 插件文件使用类方法装饰器

```python
# plugins/my_plugin.py

from easybot import Plugins, CommandValidScenes

@Plugins.on_command(command="/help")
async def help_cmd(msg):
    await msg.reply("这是帮助信息")

@Plugins.before_command(valid_scenes=CommandValidScenes.ALL)
async def log_message(msg):
    print(f"[日志] {msg.content}")
```

### 5. 单文件程序使用实例方法装饰器

```python
# main.py

from easybot import Bot

bot = Bot(app_id="xxx", app_secret="xxx")

@bot.on_command(command="/ping")
async def ping(msg):
    await msg.reply("Pong!")

@bot.before_command()
async def log(msg):

    return True
```

## 热重载插件

EasyBot SDK 支持插件热重载，无需重启机器人即可更新插件代码。

### 通过 Bot 实例操作

```python
# 热重载单个插件
result = bot.reload_plugin("admin")
print(f"重载结果: {result}")

# 热重载所有插件
results = bot.reload_all_plugins()

# 卸载插件
bot.unload_plugin("admin")

# 获取已加载插件列表
plugins = bot.get_loaded_plugins()
```

### 通过 Plugins 类操作

```python
from easybot import Plugins

# 热重载指定插件（支持插件名或命令名）
result = Plugins.reload_plugin("admin")

# 卸载插件
Plugins.unload_plugin("admin")

# 获取已加载插件列表
plugins = Plugins.get_loaded_plugins()

# 获取插件注册的命令
commands = Plugins.get_plugin_commands("admin")
```

### 热重载返回值

```python
{
    "module": "admin",           # 模块名
    "unloaded": {                # 卸载的内容
        "commands": 2,
        "preprocessors": 1
    },
    "loaded": {                  # 新加载的内容
        "commands": 2,
        "preprocessors": 1
    },
    "success": True,             # 是否成功
    "error": None                # 错误信息（如果失败）
}
```

## Bot 实例插件管理方法

Bot 类提供了完整的插件管理代理方法：

### 命令查询

| 方法 | 说明 |
|------|------|
| `get_all_commands()` | 获取所有命令对象（包括禁用的） |
| `find_command(func_name)` | 根据函数名查找命令对象 |

### 命令控制

| 方法 | 说明 |
|------|------|
| `enable_command(func_name)` | 启用指定命令 |
| `disable_command(func_name)` | 禁用指定命令 |
| `is_command_enabled(func_name)` | 检查命令是否启用 |
| `remove_command(func_name)` | 移除指定命令 |

### 插件管理

| 方法 | 说明 |
|------|------|
| `get_loaded_plugins()` | 获取已加载插件列表 |
| `get_plugin_commands(plugin_name)` | 获取插件的命令列表 |
| `get_plugin_preprocessors(plugin_name)` | 获取插件的预处理器列表 |
| `reload_plugin(name)` | 热重载指定插件（支持插件名或命令名） |
| `reload_all_plugins()` | 热重载所有插件 |
| `unload_plugin(plugin_name)` | 卸载指定插件 |
| `clear_all_plugins()` | 清空所有命令和预处理器 |

### 使用示例

```python
# 查询命令
commands = bot.get_all_commands()
for cmd in commands:
    print(f"{cmd.func.__name__}: enabled={cmd.enabled}")

# 动态控制命令
bot.disable_command("ping_cmd")    # 临时禁用
bot.enable_command("ping_cmd")     # 重新启用

# 检查状态
if bot.is_command_enabled("ping_cmd"):
    print("命令已启用")

# 移除命令
bot.remove_command("old_cmd")

# 查看插件信息
plugins = bot.get_loaded_plugins()
for plugin in plugins:
    cmds = bot.get_plugin_commands(plugin)
    print(f"{plugin}: {cmds}")
```
