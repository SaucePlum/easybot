# 常见问题 Q&A

汇总 EasyBot SDK 开发过程中可能遇到的问题及解决方案。

---

## 安装与配置

### Q1: 安装后导入报错 `ModuleNotFoundError`

**现象**:
```
ModuleNotFoundError: No module named 'easybot'
```

**原因**: 未正确安装包或安装到了错误的 Python 环境。

**解决方案**:
```bash
# 确认使用正确的 Python
python --version

# 在项目目录下安装（开发模式）
cd f:\开发\Python\easybot
pip install -e .

# 或直接安装
pip install -r requirements.txt

# 验证安装
python -c "from easybot import Bot; print('OK')"
```

### Q2: 运行时提示缺少依赖

**现象**:
```
ModuleNotFoundError: No module named 'aiohttp'
```

**解决方案**:
```bash
pip install aiohttp>=3.9.0
```

> **注意**: EasyBot SDK 依赖 `aiohttp`（HTTP/WebSocket 通信）和 `pyyaml`（插件配置）。确保 aiohttp 版本 ≥ 3.9.0。

### Q3: AppID 和 AppSecret 在哪里获取？

**步骤**:
1. 访问 [QQ 开放平台](https://q.qq.com/)
2. 登录并进入「机器人」管理页面
3. 创建或选择你的机器人应用
4. 在「开发设置」中查看 **AppID** 和 **AppSecret**

> ⚠️ **安全提示**: 不要将 AppSecret 提交到公开仓库。建议使用环境变量或 `.env` 文件管理。

```python
import os
from easybot import Bot

bot = Bot(
    app_id=os.environ["BOT_APP_ID"],
    app_secret=os.environ["BOT_APP_SECRET"],
)
```

## 连接问题

### Q4: WebSocket 连接失败 / 超时

**现象**:
```
WebSocket connection failed: timeout
Connection error: Unable to connect to gateway
```

**排查步骤**:

1. **检查网络连通性**
   ```bash
   # 测试能否访问 QQ 开放平台网关
   curl https://sandbox.api.sgroup.qq.com/websocket/gateway
   ```

2. **检查是否需要代理**
   ```python
   import os
   os.environ["HTTP_PROXY"] = "http://127.0.0.1:7890"
   os.environ["HTTPS_PROXY"] = "http://127.0.0.1:7890"
   ```

3. **确认沙箱/正式环境**
   - 沙箱环境 URL: `sandbox.api.sgroup.qq.com`
   - 正式环境 URL: `api.sgroup.qq.com`
   - 使用 `is_sandbox=True` 切换沙箱模式

4. **调整超时时间**
   ```python
   from easybot import Proto
   
   bot = Bot(
       app_id="xxx",
       app_secret="xxx",
       protocol=Proto.websocket(connect_timeout=60.0),
   )
   ```

### Q5: Webhook 模式下无法接收事件

**排查清单**:

| 检查项 | 说明 |
|--------|------|
| 端口是否被占用 | `netstat -ano \| findstr :8080` |
| 公网可达性 | 使用 ngrok/frp 等工具暴露本地端口 |
| QQ 开放平台回调地址 | 必须填写公网可访问的 URL |
| SSL 证书 | 生产环境要求 HTTPS |

**示例 — 本地开发使用 Webhook**:
```bash
# 安装 ngrok 或类似工具
ngrok http 8080
# 获得 https://xxxx.ngrok-free.app

# 设置 Bot
from easybot import Bot, Proto

bot = Bot(
    app_id="xxx",
    app_secret="xxx",
    protocol=Proto.webhook(port=8080, path="/callback"),
)

# 在 QQ 开放平台设置:
# 回调地址 → https://xxxx.ngrok-free.app/callback
```

### Q6: Remote Webhook 断连频繁

**原因**: 心跳超时或网络不稳定。

**调整参数**:
```python
from easybot import Proto

bot = Bot(
    app_id="xxx",
    app_secret="xxx",
    protocol=Proto.remote_webhook(
        url="wss://your-relay-server.com/ws",
        heartbeat_interval=30.0,     # 缩短心跳间隔
        no_msg_timeout=300.0,        # 增加无消息断线阈值
        connect_timeout=20.0,
    ),
)
```

## 消息收发

### Q7: 收不到频道消息

**常见原因及解决**:

| 原因 | 解决方案 |
|------|---------|
| 未订阅 Intent | 确保 `on_guild_message` 已注册（SDK 自动处理） |
| 公域未 @机器人 | 公域机器人只能接收 @它的消息 |
| 私域未开启全量消息 | 私域需在开放平台开启 GUILD_MESSAGES Intent |
| 沙箱过滤 | 检查 SandBox 配置是否排除了该频道 |

**验证方式**:
```python
@bot.on_guild_message
async def debug_msg(msg):
    print(f"收到消息: {msg.content}")  # 看控制台是否有输出
```

### Q8: 发送消息失败 — 权限不足

**现象**:
```
API Error: 403 Forbidden
权限不足，无法发送消息
```

**解决方案**:

1. **检查子频道发言权限**
   ```python
   perms = await bot.api.get_channel_user_permissions(channel_id=ch_id, user_id=bot.bot_id)
   print(perms.permissions)  # 查看权限位图
   ```

2. **授予机器人发送消息权限**
   - 进入频道 → 子频道设置 → 权限管理
   - 为机器人角色添加「发送消息」权限

3. **使用私域机器人**
   - 私域机器人默认拥有更多权限
   - 在开放平台将机器人设为私域：`is_private=True`

### Q9: 发送图片不显示

**排查步骤**:

1. **图片 URL 是否可访问**
   - 必须是公网可访问的 HTTPS URL
   - QQ 会下载图片进行审核和转存

2. **使用 file_image 发送本地图片**
   ```python
   await bot.api.send_guild_message(
       channel_id="xxx",
       content="图片",
       file_image="./image.png",      # 支持文件路径
   )
   
   # 或使用 bytes
   with open("./image.png", "rb") as f:
       await bot.api.send_guild_message(
           channel_id="xxx",
           content="图片",
           file_image=f.read(),         # 支持 bytes
       )
   ```

3. **群聊/单聊场景发送图片**
   ```python
   # 群聊/单聊不支持直接传 file_image，需先上传获取 file_info
   file_info = await bot.api.upload_media(
       file_type=1,  # 1=图片, 2=视频, 3=语音, 4=文件
       file_data="./image.png",  # 支持 bytes 或文件路径
       group_openid="xxx",       # 群聊用 group_openid，单聊用 user_openid
   )
   
   # 使用 media_file_info 发送
   msg = MessagesModel.Message(
       content="图片消息",
       media_file_info=file_info.file_info,
   )
   await bot.api.send_group_message(group_openid="xxx", content=msg)
   ```

### Q10: Embed/Ark 消息格式异常

**常见问题**:

- **Embed 字段过多**: 单个 Embed 最多约 25 个字段
- **Markdown 渲染异常**: QQ 平台不支持所有 Markdown 语法
- **Ark 模板 ID 错误**: template_id 必须从开放平台获取

**调试方法**:
```python
embed = MessagesModel.MessageEmbed(title="测试", content=["行1", "行2"])
print(embed.build())  # 打印生成的 JSON 结构
```

## 命令系统

### Q11: 注册的命令没有被触发

**排查清单**:

1. **命令词是否完全匹配**
   ```python
   # ❌ 用户输入 "/Ping" 但注册的是 "/ping" (大小写敏感)
   @bot.on_command(command="/ping")  # 只匹配小写
   
   # ✅ 解决方案: 同时注册多种写法
   @bot.on_command(command=["/ping", "/Ping", "/PING"])
   ```

2. **is_treat 导致内容为空**
   ```python
   @bot.on_command(command="/echo")
   async def echo(msg):
       # 如果用户只输入了 "/echo"，treated_msg 为空字符串 ""
       content = msg.treated_msg or "(无内容)"
       await msg.reply(content)
   ```

3. **valid_scenes 场景限制**
   ```python
   # 默认 valid_scenes 为 ALL（全场景），如需限制特定场景：
   @bot.on_command(command="/guild_only", valid_scenes=CommandValidScenes.GUILD)  # 仅频道
   @bot.on_command(command="/group_only", valid_scenes=CommandValidScenes.GROUP)  # 仅群聊
   ```

4. **预处理器中断了流程**
   ```python
   @bot.before_command()
   async def pre(msg):
       if some_condition:
           raise StopProcessing()  # 这会导致后续命令不被执行
   ```

### Q12: 正则命令匹配不到

**常见错误**:

```python
# ❌ 忘记 ^ 和 $ 导致部分匹配
@bot.on_command(regex=r"计算\s+\d+")
# 输入 "帮我计算 123" 不会匹配（因为前面有其他文字）

# ✅ 使用 ^ 匹配开头（配合 is_treat=True）
@bot.on_command(regex=r"^计算\s+(.+)$")

# ❌ 正则中有特殊字符未转义
@bot.on_command(regex=r"/cmd\?id=\d+")  # ? 是正则特殊字符
# ✅ 转义
@bot.on_command(regex=r"/cmd\?id=(\d+)")
```

### Q13: 如何实现命令别名？

```python
# 方式一: command 参数传入列表
@bot.on_command(command=["/help", "/h", "帮助"])
async def help_cmd(msg):
    await msg.reply("帮助信息...")

# 方式二: 多个装饰器绑定同一函数
def handle_ping(msg):
    return msg.reply("Pong!")

bot.on_command("/ping")(handle_ping)
bot.on_command("/p")(handle_ping)
```

## API 调用

### Q14: API 返回错误码

**常见错误码**:

| 错误码 | 含义 | 解决方案 |
|--------|------|---------|
| 40001 | access_token 无效 | 检查 AppSecret 是否正确 |
| 40002 | access_token 过期 | SDK 自动刷新，无需手动处理 |
| 40303 | 权限不足 | 检查机器人权限配置 |
| 42900 | 请求过于频繁 | 降低调用频率，添加限流 |
| 50001 | 内部服务器错误 | 稍后重试 |

**查看详细错误信息**:
```python
try:
    guild = await bot.api.get_guild(guild_id="xxx")
except Exception as e:
    print(f"API 调用失败: {e}")
    # SDK 默认会记录错误日志（如果 is_log_error=True）
```

### Q15: API 调用超时

**原因**: 网络延迟或服务器响应慢。

**解决方案**:
```python
# 方式一: 增加 API 超时时间
bot = Bot(app_id="xxx", app_secret="xxx", api_timeout=60)

# 方式二: 配置重试次数
bot = Bot(app_id="xxx", app_secret="xxx", is_retry=5)
```

### Q16: 分页数据如何完整获取？

```python
async def get_all_members(bot, guild_id):
    """获取频道全部成员"""
    all_members = []
    after = "0"
    
    while True:
        members = await bot.api.get_guild_members(
            guild_id=guild_id,
            after=after,
            limit=100,
        )
        if not members:
            break
        all_members.extend(members)
        after = members[-1].user.id
    
    return all_members
```

## 会话管理

### Q17: Session 数据丢失

**可能原因**:

1. **超时过期** — 设置了 `timeout` 且长时间未访问，会话状态变为 INACTIVE
2. **作用域/标识不匹配** — `new()` 和 `get()` 使用了不同的 `scope` 或 `key`
3. **程序重启后未恢复** — 虽然 SDK 支持自动持久化（pickle），但需确认 `commit_path` 可写

**调试方法**:
```python
from easybot import Scope

with bot.session.bind(msg) as s:
    session = s.get(scope=Scope.USER, key="my_key")
    if session and session.status == 0:
        print(f"会话数据: {session.data}")
    else:
        print("会话不存在或已失活")
```

### Q18: 如何持久化 Session？

EasyBot 的 SessionManager **内置自动持久化功能**（基于 pickle 文件）：

- 默认存储路径: `<工作目录>/sdk_data/sessions.pickle`
- 每次 `new()` / `update()` / `remove()` 后自动写入
- Bot 启动时自动从文件恢复

```python
# 如需自定义持久化路径，可在创建 Bot 后修改:
# bot.session = SessionManager(logger=logger, commit_path="./custom_path")
```

> **注意**: 持久化使用 Python pickle 格式，适合单机部署。如需跨进程/分布式共享 Session，需要自行实现外部存储后端。

### Q19: 并发请求导致 Session 冲突

**场景**: 同一用户快速发送多条消息，可能导致读写冲突。

**缓解措施**: `get()` 和 `update()` 操作本身是同步的（非异步），在高并发场景下建议：
```python
from easybot import Scope

# 先获取再更新，尽量减少中间状态时间
with bot.session.bind(msg) as s:
    session = s.get(scope=Scope.USER, key="my_key")
    if session and session.status == 0:
        new_data = dict(session.data)
        new_data["counter"] = new_data.get("counter", 0) + 1
        s.update(scope=Scope.USER, key="my_key", data=new_data)
```

## 性能与部署

### Q20: 内存占用过高

**优化建议**:

1. **合理设置 timeout 和 inactive_gc_timeout**
   ```python
   from easybot import Scope

   # 减少超时时间
   with bot.session.bind(msg) as s:
       s.new(scope=Scope.USER, key="temp", data={...}, timeout=120)

   # 及时清理无用会话（remove 不需要 bind）
   bot.session.remove(scope=Scope.USER)
   ```

2. **避免在事件处理器中存储大量数据**
   ```python
   # ❌ 不好的做法 — 把整个聊天记录存在 session 里
   # session.data["messages"].append(large_message_object)

   # ✅ 好的做法 — 只存必要的状态
   with bot.session.bind(msg) as s:
       s.new(scope=Scope.USER, key="chat", data={
           "last_msg_id": msg.id,
           "step": "waiting_reply",
       })
   ```

3. **使用生产级 WSGI/ASGI 服务器**
   ```bash
   # 开发环境
   python main.py
   
   # 生产环境 — 使用 uvicorn
   uvicorn main:app --host 0.0.0.0 --port 8080 --workers 4
   ```

### Q21: 如何部署到 Linux 服务器？

**推荐方案**:

1. **使用 systemd 管理**
   ```ini
   # /etc/systemd/system/easybot.service
   [Unit]
   Description=EasyBot Robot
   After=network.target
   
   [Service]
   Type=simple
   User=www-data
   WorkingDirectory=/opt/easybot
   ExecStart=/usr/bin/python3 main.py
   Restart=always
   RestartSec=5
   
   [Install]
   WantedBy=multi-user.target
   ```
   
   ```bash
   sudo systemctl enable easybot
   sudo systemctl start easybot
   sudo journalctl -u easybot -f  # 查看日志
   ```

2. **使用 Docker**
   ```dockerfile
   FROM python:3.11-slim
   WORKDIR /app
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   COPY . .
   CMD ["python", "main.py"]
   ```

3. **使用 Supervisor**
   ```ini
   [program:easybot]
   command=python main.py
   directory=/opt/easybot
   autostart=true
   autorestart=true
   stdout_logfile=/var/log/easybot.log
   stderr_logfile=/var/log/easybot_error.log
   ```

## 其他问题

### Q23: 如何同时监听多个事件类型？

```python
# 一个处理器可以注册多个事件
async def log_all(event_data):
    print(f"收到事件: {type(event_data).__name__}")

bot.on_guild_message(log_all)
bot.on_group_message(log_all)
bot.on_c2c_message(log_all)
```

### Q24: 如何优雅关闭机器人？

```python
# 方式一: Ctrl+C（自动捕获 KeyboardInterrupt）
bot.start()

# 方式二: 异步上下文管理器
async def main():
    async with bot:
        await bot.start_async()
        # ... 业务逻辑 ...
    # 退出代码块时自动清理

# 方式三: 手动停止
await bot.stop_async()
```

### Q25: 如何获取机器人的用户 ID？

```python
# 启动前: bot.bot_id 为 None
print(bot.bot_id)  # None

# 启动后: 自动填充
bot.start()

# 或在启动事件中获取
@bot.on_startup
async def on_ready(event):
    me = await bot.api.get_me()
    print(f"机器人 ID: {me.id}, 名称: {me.username}")
```
